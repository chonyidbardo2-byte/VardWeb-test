WCAG
Compliance Report

A transparent, criterion-by-criterion record of our conformance to the Web Content Accessibility Guidelines 2.1. Honest about where we pass. Honest about where we're still improving.

Last evaluated: June 2026

// 01 — scope of this report

What this report covers.

Covered URLs: All pages on vardweb.ca and vardweb.com, including the main landing page, About, FAQ, Blog, Contact, Our Work, and all legal pages.

Standard: Web Content Accessibility Guidelines (WCAG) version 2.1, published by the W3C Web Accessibility Initiative. Reference: w3.org/TR/WCAG21

Target Level: AA (including all Level A criteria)

Date of last evaluation: June 2026. This report is reviewed and updated at minimum annually, and whenever substantial changes are made to site content or functionality.

Relied-upon technologies: HTML5, CSS3, JavaScript (ES Modules), WAI-ARIA 1.2

// 02 — conformance claim

Our formal accessibility declaration.

📜

VardWeb declares partial conformance with WCAG 2.1 Level AA for vardweb.ca and vardweb.com. "Partial conformance" means that some content does not fully conform to the standard at the date of this report. Known non-conformances are documented in Section 10.

This conformance claim is made in accordance with the W3C's WCAG-EM (Evaluation Methodology) guidance. The claim covers all pages and page states accessible without authentication. Authenticated areas (client portal) are subject to a separate internal review.

// 08 — known exceptions & non-conformances

What we know. What we're doing about it.

2.4.1 — Skip Navigation (Level A)
A "Skip to main content" link is absent from all pages. Users who navigate by keyboard must tab through the entire navigation bar to reach the main content on each page. Remediation plan: Add a visually hidden focusable skip link as the first element in the <body> of all page templates. Target: Q3 2026.

2.2.2 — Reduced Motion (Level A, partial failure)
CSS animations (hero gradient pulses, button hover transforms, footer heartbeat) do not currently pause or reduce when the user has set prefers-reduced-motion: reduce in their operating system. This may cause discomfort for users with vestibular disorders. Remediation plan: Wrap all animation declarations in @media (prefers-reduced-motion: no-preference) blocks. Target: Q3 2026.

1.1.1 — Decorative Icons (Level A, minor)
Some inline SVG icons used decoratively lack explicit aria-hidden="true", causing screen readers to announce meaningless characters. Remediation plan: Audit and label all inline SVGs as decorative or informative. Target: Q3 2026.

4.1.3 — Status Messages (Level AA)
Form submission confirmations are not currently announced to screen readers via ARIA live regions. Users relying on screen readers may not receive feedback upon form submission. Remediation plan: Add aria-live="polite" regions for all form success and error states. Target: Q3 2026.

// 09 — testing methodology

How this report was produced.

Automated Testing

Pages were tested with axe-core 4.x via the axe browser extension and integrated into the development workflow. WAVE Evaluation Tool was used for structure and label review. Google Lighthouse Accessibility Audit was run on all key page templates. Colour contrast was verified with WebAIM's Contrast Checker for all text/background combinations.

Manual Testing

Full keyboard-only traversal was performed on all pages using Chrome on Windows 10 and Safari on macOS. Screen reader testing was conducted using NVDA 2024.x + Chrome and VoiceOver + Safari. ARIA roles and live region behaviour were verified manually. Focus order and trapping were tested on all interactive components including the mobile accordion navigation, dropdown menus, and form controls.

Scope of Review

This report covers public-facing pages accessible without authentication. Client portal pages (under /crm/) are subject to a separate, ongoing internal review and are not covered by this public conformance claim.
